Overloads
*Cafe*
 - sellCoffee(int size, int nSugarPackets, int nCreams, int cups): sells multiple of the same order of coffee at a time
 - sellCoffee(): sells one 8oz cup of black coffee
 I overloaded these two because 1. People order multiple cups of coffee sometimes and 2. I feel like there should be a regular baseline black coffee to order and anything else is an add on

*Library*
 - containsTitle(ArrayList<String> titles): checks to see if all the books in a list are available
 - isAvailable(ArrayList<Boolean> titles): checks to see which books in a list are available and which aren't
 I overloarded these two because I know that when I was little I would checkout a bunch of books at a time, and in theory if I felt like dedicating more time to this, I would have also written out a checkout/checkin method that took a list of books.

 *House*
 - House(): builds a default house with default given values, nothing input
 - House(String name, String address, int nFloors, boolean diningRoom, boolean elevator, ArrayList<Student> residents): builds a house that already has residents living in it
 I overloaded these because sometime houses already have people living in them, and sometimes you know literally nothing about a house and both are valid.
